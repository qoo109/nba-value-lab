# G1.2.0 EV-Primary — 2026-27 Season Activation Decision

Updated: 2026-07-23  
Project: NBA Value Lab  
Formal Stake: `0`

## Decision

```text
G1_2_0_EV_PRIMARY_USER_APPROVED_FOR_2026_27_SEASON
```

使用者已批准：下一個 NBA 例行賽季開始後，G 系統改以 EV-primary 新版作為主要輸出。

正式版本名稱採用：

```text
G1.2.0-20260723
```

原因是現行正式基準已是 `G1.1.1-20260719`；舊版本不可覆寫，因此不能再將新版命名為 G1.1。

## 新賽季執行方式

```text
V3.1.1：維持不變
G1.2.0：新賽季主要 G 輸出
G1.1.1：平行 Control／Shadow
Coordination：同一 cutoff、同一資料版本、兩引擎獨立
Stake：0
```

啟用觸發：

```text
first_2026_27_regular_season_game_with_complete_T60_data_gate
```

若第一批比賽的資料 Gate 未完整，系統必須輸出 `資料不足`，不得為了強行產生選場而放寬 Gate。

## EV-primary 固定門檻

```text
最低候選 EV：5%
ㄇ級：5% 至未滿 7%
ㄆ級：7% 至未滿 10%
ㄅ級：10% 以上
PP Edge 最低安全線：2pp
```

EV 計算：

```text
ConservativeEV = P_C × Decimal Odds − 1
```

其中 `P_C` 必須是價格鎖定前的保守模型勝率；目標莊家價格不得回寫模型勝率。

## 保留的硬 Gate

新版只更換價值判斷主指標，不移除：

- Coverage `>=85%`
- 傷病與輪替確認
- 高信心與區間寬度限制
- 新聞風險限制
- 至少 3 個比較來源
- 同莊家、同市場、同時間的雙邊價格
- OOD、stale price 與未建模反向路徑阻擋
- 雙邊價值衝突阻擋主要場次
- T-60 鎖定、T-5 重選、Closing 僅驗證
- 主要場次目標 2、上限 3、允許 0

## Control 規則

G1.1.1 在新賽季仍必須平行計算並保存，但：

- 不主導當日 G 主要輸出；
- 不覆蓋 G1.2.0 的分級；
- 僅用於比較選場重疊、ROI、CLV、回撤與穩定性；
- 負結果也必須保存，不得事後只保留有利區間。

## 不代表的事項

本決策不代表：

- EV-primary 已經證明能獲利；
- 允許真金投注；
- 允許提高 Stake；
- 允許模型重訓或機率融合；
- 允許使用缺乏 point-in-time 身分的 Closing-only 資料冒充可執行價格。

正式 Stake 持續為 `0`。
