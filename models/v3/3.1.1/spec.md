## 角色與物件邊界

V 負責比較已鎖定的機率與目標莊家價格。`prediction` 與 `price_evaluation` 必須分離：

- 價格變動不得改寫 `P_F/P_M/P_C/P_N/P_O` 或 `prediction_id`。
- 每次有效價格變動新增不可覆寫的 `price_evaluation_id`。
- 目標莊家價格不得進入任何會改變勝率的模型特徵。
- `P_C/P_N/P_O` 由上游模型產生；V 不用目標價格回推或人工調整勝率。

## 共用價格數學

機率在資料與 Python 紀錄中使用 0～1；畫面百分比與 pp 顯示乘以 100。十進位賠率記為 `O`，對手賠率記為 `O_opp`。

| 輸出 | 公式 |
|---|---|
| 損益平衡機率 `P_BE` | `1 / O` |
| 雙向比例去水機率 | `(1 / O) / ((1 / O) + (1 / O_opp))` |
| 情境 EV | `P * O - 1` |
| 保守優勢 `EdgePP` | `(P_C - P_BE) * 100` |
| 距離門檻 `ThresholdDistancePP` | `EdgePP - RequiredMarginPP` |
| 最低接受賠率 | `1 / (P_C - RequiredMarginPP / 100)` |

最低接受賠率只有在分母大於 0 時定義。去水方法固定為雙向比例法；更換去水方法視為模型規則變更，必須建立新版本並重新驗證。

## 價格政策

`core_odds_scope = [1.40, 1.60]` 只表示 V 可取得 ㄅ級的核心區，不代表 V 的全部可分類範圍。

| 程式邊界 | 定位 | 未獨立驗證前最高結論 |
|---|---|---|
| `[1.20, 1.30)` | 極低價格區 | 排除・極低價格 |
| `[1.30, 1.40)` | 低價延伸研究區 | ㄆ級・延伸研究 |
| `[1.40, 1.60]` | 核心決策區 | 可依完整規則列 ㄅ級・研究候選 |
| `(1.60, 1.75]` | 高價延伸研究區 | ㄆ級・延伸研究 |
| `(1.75, 2.00)` | 另行校準區 | 另行校準 |
| 其餘，包括 `2.00` | 超出目前策略 | 範圍外 |

延伸區只封頂、不自動升級；若原始數學結果更低，維持較低結論。

## 基礎分級表

規則依序執行，後面的規則不得把較低結論升高。

| 條件 | 基礎分級 |
|---|---|
| 關鍵資料不足、信心為資料不足或新聞風險 3 | 資料不足 |
| `EdgePP < 0` | 不支持 |
| `RequiredMarginPP` 不適用 | ㄆ級，再套用價格區封頂 |
| `ThresholdDistancePP >= 0` | ㄅ級 |
| `-3 <= ThresholdDistancePP < 0` | ㄆ級・條件觀察 |
| `ThresholdDistancePP < -3` 且 `EdgePP >= 0` | ㄇ級・價格合理 |

新聞風險 2 或低信心會把 ㄅ級降為 ㄆ級。價格區、研究階段與協調層可以繼續向下降級或封頂，不能向上升級。

## 資料與驗證限制

- Coverage < 50% 或關鍵欄位失敗：資料不足。
- 風險 2：最高 ㄆ級；風險 3：停止正式價格分級。
- 禁止隨機切割，使用 nested walk-forward，final holdout 只解封一次。
- 未通過 Stage 2、point-in-time 盤口與前瞻驗證前，所有候選均為研究輸出，正式投注額固定為 0。

## 完整性驗證

`config.json` 的 `spec_integrity` 對本檔 UTF-8 精確檔案位元組計算 SHA-256，不做換行、空白或 Unicode 正規化。驗證器必須在 Registry 發布前重算並比對。
