## 核心變更

1. 模型預測 `prediction` 與價格評估 `price_evaluation` 完全分離。
2. 單純價格變動不得改寫 `P_F/P_M/P_C/P_N/P_O`、安全邊際、最低接受賠率或 `prediction_id`。
3. 每次有效價格變動新增不可覆寫的 `price_evaluation_id`。
4. 目標莊家當前價格、價格區間、價差與價格年齡不得進入任何會改變勝率的模型。
5. 人工覆核採盲化流程；在覆核鎖定前隱藏價格、EV、最低接受賠率與分級。
6. 公開新聞必須檢查市場是否已先反應；新聞本身不等於新資訊優勢。
7. Coverage 改為「通過的適用欄位檢查數 ÷ 全部適用欄位檢查數」，關鍵欄位不可用其他欄位補償。
8. Stage 0～2 固定安全邊際為 5pp；Stage 3 動態邊際必須另行註冊與驗證。

## 價格政策

| 程式邊界 | 定位 | 未獨立驗證前最高結論 |
|---|---|---|
| `[1.20, 1.30)` | 極低價格區 | 排除・極低價格 |
| `[1.30, 1.40)` | 低價延伸研究區 | ㄆ級・延伸研究 |
| `[1.40, 1.60]` | 核心決策區 | 可依完整規則列 ㄅ級・研究候選 |
| `(1.60, 1.75]` | 高價延伸研究區 | ㄆ級・延伸研究 |
| `(1.75, 2.00)` | 另行校準區 | 另行校準 |
| 其餘 | 超出目前策略 | 範圍外 |

延伸區只封頂、不自動升級；若原始數學結果更低，維持較低結論。

## 分級優先序

1. 模型資料不足。
2. 價格資料不足。
3. 價格範圍閘門。
4. `P_C < P_BE`：模型不支持。
5. 依 `ThresholdDistance = P_C - P_BE - RequiredMargin` 建立基礎等級。
6. 依信心、新聞風險、階段及延伸區規則向下降級或封頂。

後面的規則不得把前面的低等級升高。

## 風險與資料限制

- 風險 0～1：其他條件完整時可進入核心判定。
- 風險 2：最高 ㄆ級。
- 風險 3：未確認前停止正式價格分級。
- 低信心：最高 ㄆ級。
- 關鍵欄位失敗或 Coverage < 50%：資料不足。

## 驗證治理

- 禁止隨機切割。
- 使用 nested walk-forward。
- final holdout 只解封一次。
- 必須保存 `validation_run_id`、模型與資料版本、原始與有效樣本數、全部指標及不確定區間。
- 未通過 Stage 2 前，任何 ㄅ級都只屬研究候選，正式投注額固定為 0。

## 儲存原則

GitHub 保存程式、規格、Schema、測試與小型 fixture；不得保存秘密、大型歷史資料庫、全量高頻盤口、大量原始 HTML/PDF 或未確認保存權的資料。
